# GA12008_assignment
<h2>Assignment 1 for module GA12008 - Information Systems Development &amp; Management</h2>

<h3>Overview</h3>

<p>This project is a web-interface for the fiction company Seal Energy's Customer Information Portal. The portal is designed to provide customers with clear information about energy services, tariffs, contact options, and personal account details.</p>

<p>This project is built using HTML5, CSS and Google Fonts. </p>

<h3>Features</h3>

<h4>Homepage</h4>
<ul>
  <li>Hero banner with strong branding and call‑to‑action</li>
  <li>Overview of services and features</li>
  <li>Responsive grid layout for feature cards</li>
</ul>
  

<h4>Information Page</h4>
<ul>
  <li>Clear, structured content sections</li>
  <li>Card‑based layout for readability</li>
  <li>Consistent typography and spacing</li>
</ul>

<h4>Profile Page</h4>
<ul>
  <li>User welcome message</li>
  <li>Account Information and Current Tariff displayed as responsive cards</li>
  <li>Energy usage table with alternating row colours and responsive scrolling</li>
</ul>

<h4>Contact Page</h4>
<ul>
  <li>Two‑column layout with equal‑width cards</li>
  <li>Contact details and a fully styled contact form</li>
  <li>Mobile‑friendly stacked layout</li>
</ul>

<h4>Design System</h4>
<ul>
  <li>CSS variables for colours, fonts, spacing, and border radius</li>
  <li>Reusable card components</li>
  <li>Consistent button styling</li>
  <li>Rem‑based spacing for improved accessibility</li>
</ul>

<h4>Technologies Used</h4>
<ul>
  <li>HTML5 – semantic structure</li>
  <li>CSS – custom styling, responsive layout, CSS variables</li>
  <li>Google Fonts – Quicksand and Nunito</li>
  <li>Flexbox & CSS Grid – layout and responsiveness</li>
</ul>


<h4>Responsive Design</h4>
The site is fully responsive and adapts to:
<ul>
  <li>Mobile phones</li>
  <li>Tablets</li>
  <li>Desktop screens</li>
</ul>


Key responsive features:
<ul>
  <li>Navigation spacing adjusts at smaller widths</li>
  <li>Contact page columns collapse into a single column</li>
  <li>Profile cards stack vertically on mobile</li>
  <li>Tables scroll horizontally when needed</li>
</ul>


<h4>Accessibility Considerations</h4>
<ul>
  <li>Semantic HTML structure</li>
  <li>Proper heading hierarchy</li>
  <li>Accessible form labels</li>  
  <li>High‑contrast colour palette</li>
  <li>Focus‑visible states on interactive elements</li>
  <li>Screen‑reader‑friendly content structure</li>
</ul>
